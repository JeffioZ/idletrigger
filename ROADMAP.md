## UI/UX 后续计划：Quiet Native Utility

本节只记录尚未实施、且经当前 Win32/GDI 实现复核后仍有价值的 UI/UX 工作。原则是优先真实可用性、无障碍和维护性；不改变既有信息架构，不引入 WebView、网页技术或大型 UI 框架。

### 已完成（不再列为待办）

- [x] popup 资源生命周期、style / metrics / layout 集中，以及真实深浅色、中英文 UI 截图基线。
- [x] choice button + choice layer、触发器上下箭头和真实 open 状态、selected marker 与稳定字重。
- [x] 键盘 focus-visible 语义与圆角 focus ring、Toggle 的局部 hover / pressed 反馈、Shutdown / Restart 的危险分组与分隔线。
- [x] 主图标、深浅 tray/title 图标和 GitHub Social Preview 的统一品牌资源更新。

### 近期推荐

- [ ] **Windows High Contrast 支持（推荐实施；高优先级）**
  - **目标与价值：** 检测系统高对比度，并把 owner-draw palette 映射到 Windows 系统色；让边框、focus、selected、checked、disabled 和 danger 在辅助主题下仍可辨识，收益覆盖所有键盘和低视力用户。
  - **实施建议与前置：** 以现有语义 palette 为入口，使用 `SPI_GETHIGHCONTRAST` 及系统色生成专用 token；主题/DPI 刷新时重算。保留 marker、勾选 glyph 与 focus ring，不能只依赖颜色。
  - **成本、风险与阶段：** 中等成本，风险集中在自绘控件和截图差异；先在 popup 建立最小模式，再人工验证高对比度浅/深主题、Tab 焦点、choice selected、Toggle 与危险菜单。
  - **验收：** 不出现低对比文字或消失的边界；三种浮层、禁用控件及键盘焦点均可用，且退出高对比度后恢复普通 palette。

- [ ] **Shutdown / Restart 的原生确认（近期产品决策项；需要产品确认）**
  - **目标与价值：** 为不可逆且会中断工作的两项操作提供一次明确、默认取消的原生确认，降低误触损失；Sleep / Hibernate 继续保持快速操作。
  - **实施建议与前置：** 仅覆盖 Shutdown、Restart，使用轻量 Win32 对话框或现有 popup 体系中的明确确认文案与按钮；需确认产品仍接受“非立即执行”的交互契约。
  - **成本、风险与阶段：** 低到中等成本；风险是降低少量高级用户的快速操作效率。不要使用短暂撤销（系统电源动作不可可靠撤销），也不建议二次点击。
  - **验收：** 鼠标和键盘均能安全取消/确认；默认焦点为取消；现有命令、语言、DPI、主题和菜单关闭生命周期不回归。

### 中期候选

- [ ] **自绘语义的最小 UIA / MSAA 路径（建议分阶段实施）**
  - **目标与价值：** 补足读屏对 expanded/collapsed、selected、checked、disabled 及浮层关系的理解，而不是建立一套全面无障碍框架。
  - **实施建议与前置：** 先用 Narrator 与 Inspect.exe 实测现有原生 BUTTON/checkbox 的名称、角色、Tab 顺序和状态；仅对实测缺口增加 `WM_GETOBJECT`/provider，优先 choice trigger 的 expanded 状态、choice layer 的选择状态，再评估 quick/language 菜单关系。
  - **成本、风险与阶段：** 中到高成本，需熟悉 COM 生命周期、跨进程访问与 HWND 销毁；以真实读屏缺口为前置，不在未验证前重构全部控件。
  - **验收：** 读屏能按视觉顺序读出可操作控件、当前选择/勾选/展开状态，且 popup 隐藏、销毁、主题/DPI 切换后不残留 provider。

- [ ] **README 截图的低成本 Windows CI 证据（推荐试验后决定）**
  - **目标与价值：** 保留现有 PNG 有效性检查，并在固定 Windows 环境中捕获四张截图、作为失败或发布前的 artifact，方便发现明显视觉回归。
  - **实施建议与前置：** 固定 runner、DPI、系统字体和截图脚本版本；先上传 artifact 并人工比对，不把像素 diff 设为阻塞门禁。
  - **成本、风险与阶段：** 低到中等成本；GitHub runner 的 DPI、字体和 GDI 栅格化会造成不稳定，只有环境足够稳定时才逐步提高自动化程度。
  - **验收：** artifact 可稳定生成、尺寸与主题/语言组合正确；环境漂移不会产生无意义的阻塞失败。

### 长期研究

- [ ] **GDI+ / Direct2D 绘制后端研究（仅在有可复现质量缺陷时评估）**
  - **判断：** 当前 GDI 的控件质量与图标 supersampling 已足够，不建议为圆角抗锯齿主动迁移。
  - **收益、成本与风险：** 可能改善局部抗锯齿，但会引入 COM 初始化、设备/资源重建、截图兼容性和更多生命周期维护；需先有真实 DPI 或边缘质量问题及可量化对照。
  - **触发条件与验收：** 仅在 GDI 不能以像素对齐解决的已复现问题出现时做独立原型；原型须证明质量收益且不增加常驻资源或回归截图稳定性。

### 保持现状或仅在真实反馈后调整

| 项目 | 推荐结论 | 用户价值与实施成本 | 风险、前置与触发条件 |
| --- | --- | --- | --- |
| quick / language hover 展开与 220ms 宽限 | **保持现状，不建议近期调整** | 当前只展开菜单、不直接执行命令，open 箭头和键盘/Escape 已清晰；改为点击优先的收益未被证实。 | 误触反馈或可用性测试明确显示问题时，再单独比较点击优先方案；不能破坏现有定位和键盘规则。 |
| panel padding 18→20、section gap 14→16、统一 4/8 spacing、section title 700→600 | **暂缓；仅独立视觉试验后决定** | 数字整齐本身不是价值；当前中英文截图密度正常，机械调整会让常驻工具更高、更松。 | 若出现阅读拥挤或本地化截断证据，再做一轮固定尺寸 A/B 截图和人工验收；不与功能改动混做。 |
| Day / Night “Scheduled / 已按计划”状态 | **仅在真实理解问题出现后实施** | 可帮助少数用户理解自动主题，但在主面板新增常驻状态很可能增加噪音。 | 优先尝试短 tooltip/辅助文案；先确认当前状态提示不足，且不改变信息架构。 |
| Tooltip 视觉统一/自绘 | **不建议实施** | 当前原生 `tooltips_class32` 已随主题、DPI 和窗口生命周期工作；圆角一致性不足以抵消收益。 | 自绘会增加 DPI、无障碍、定位和销毁风险；仅修复已复现的原生 tooltip 缺陷。 |
| hover 过渡、箭头翻转、choice layer 淡入、Toggle 动效 | **不建议实施** | 静态 open/pressed/focus 状态已足够清晰，默认无动画符合安静、低资源定位。 | 动画会引入 timer、CPU、减少动画偏好和关闭生命周期问题；仅在明确可用性收益下尝试 80–120ms、可关闭且非持续计时的局部效果。 |

### 明确过滤的通用设计建议

- 不采用 Web/移动端的 44–48pt 触控目标、响应式断点、React/Tailwind/WebView、网络字体或网页式卡片布局：它们不适用于现有 Windows 鼠标/键盘原生工具。
- 不为“现代感”引入玻璃、模糊、强阴影、渐变、发光、复杂动效或独立插画；这些会削弱 Quiet Native Utility 的克制性并增加 GDI 成本。
- 不把通用“全面无障碍框架”、自绘 tooltip 或绘制后端迁移当作近期必做项；均应以实测缺口或可复现质量问题为前置。

## 后续计划

### 高优先级

- [ ] 自动升级：检查 GitHub Releases 最新版本、下载对应架构 EXE、校验 SHA256、下次启动原子替换；降低用户手动下载和覆盖 EXE 的成本
- [ ] 自动升级安全策略：仅升级稳定 tag，展示版本、大小、校验和与发布说明，失败可回滚；避免更新链路引入供应链和坏版本风险
- [ ] Windows 实机自动化冒烟测试：覆盖托盘、通知、热键、IPC、睡眠/休眠入口；减少只能靠人工回归发现的系统集成问题

### 高优先级：Windows 原生实现正统化

- [ ] 用 `PowerCreateRequest/PowerSetRequest` 替换旧的线程执行状态 API；让保持唤醒以句柄管理生命周期，减少对固定 OS 线程的依赖
- [ ] 用 `RegisterPowerSettingNotification` 监听电源与电量变化，减少 3 秒轮询，仅保留兜底检查；让电池策略和电池深色模式响应更及时、更省资源
- [ ] 用 `ReadDirectoryChangesW` 监听配置文件目录变化，替代配置自动重载的定时 `os.Stat` 轮询；让外部编辑配置后更快生效，并减少长期运行轮询开销
- [ ] 为托盘图标启用稳定 `GUID` 与 `NIM_SETVERSION`；提升 Explorer 重启、图标缓存、通知区域迁移和鼠标事件的一致性
- [ ] 评估用 Task Scheduler COM 作为开机自启后端，替代或补充 `Run` 注册表项；获得更标准的延迟启动、可管理性和系统设置可见性

### 中优先级

- [ ] CLI `status --json` 结构化输出；方便脚本、监控工具或后续自动化测试读取当前运行状态
- [ ] 自动升级 UI：浮层展示“检查更新 / 下载进度 / 重启完成升级”；让升级过程可见、可控，避免后台下载造成不确定感
- [ ] 日出日落偏移量配置；支持比实际日出更早切浅色、比日落更晚切深色等个人习惯
- [ ] 空闲时长支持自定义值；在保留常用预设的基础上允许用户输入任意分钟数

### 低优先级

- [ ] 统一 quick/language/choice 浮层的 surface metrics，移除 choice 定位中的临时像素补偿
- [ ] GDI+ / Direct2D 绘制后端：参见“UI/UX 后续计划：Quiet Native Utility”的长期研究项；仅在有可复现质量缺陷时评估
- [ ] 评估预创建 choice option HWND 的启动成本，并清理不再需要的滚动路径
- [ ] 有明确用户需求时评估独立的 Windows 7 legacy 构建；只在真实需求出现时承担 Go 1.20、旧依赖和实机验证成本
- [ ] CI 增加 race detector（仅测试增强；需配置 MinGW-w64，短期暂缓）；用于发现并发数据竞争，不影响正式 EXE
- [ ] 更多语言；在出现明确用户群体后再扩展，避免提前增加翻译维护成本
