## UI/UX 后续计划：Quiet Native Utility

本节只记录尚未实施、且经当前 Win32/GDI 实现复核后仍有价值的 UI/UX 工作。优先级只看普通用户能否直接感知并受益：减少操作成本、避免误触、让状态和设置更容易理解；不因技术完整性、无障碍覆盖率或实现“现代化”而抬高优先级。性能或架构改动也按同一标准判断：能以数据或稳定复现证明其解决卡顿、耗电、状态延迟、异常恢复等实际问题时可以前置；仅为技术纯度、抽象完整性或无障碍适配而做的重构保持低优先级。不改变既有信息架构，不引入 WebView、网页技术或大型 UI 框架。

### 已完成（不再列为待办）

- [x] choice button + choice layer、触发器上下箭头和真实 open 状态、selected marker 与稳定字重。
- [x] 键盘 focus-visible 语义与圆角 focus ring、Toggle 的局部 hover / pressed 反馈、Shutdown / Restart 的危险分组与分隔线。
- [x] 主图标、深浅 tray/title 图标和 GitHub Social Preview 的统一品牌资源更新。

### 近期推荐：直接改善日常体验

- [ ] **Shutdown / Restart 的原生确认（需要产品确认）**
  - **目标与价值：** 为不可逆且会中断工作的两项操作提供一次明确、默认取消的原生确认，降低误触损失；Sleep / Hibernate 继续保持快速操作。
  - **实施建议与前置：** 仅覆盖 Shutdown、Restart，使用轻量 Win32 对话框或现有 popup 体系中的明确确认文案与按钮；需确认产品仍接受“非立即执行”的交互契约。
  - **成本、风险与阶段：** 低到中等成本；风险是降低少量高级用户的快速操作效率。不要使用短暂撤销（系统电源动作不可可靠撤销），也不建议二次点击。
  - **验收：** 鼠标和键盘均能安全取消/确认；默认焦点为取消；现有命令、语言、DPI、主题和菜单关闭生命周期不回归。

- [ ] **关键状态与失败原因的轻量反馈**
  - **用户价值：** 用户执行设置或等待自动行为后，能知道是否已生效、为何暂未生效，以及下一步该做什么；优先覆盖更新失败、主题切换因全屏暂停、保持唤醒与空闲监测互斥等已有概念。
  - **交互边界：** 只在用户主动操作后、操作失败时或确实需要处理时显示短提示；成功时不堆积通知，不把紧凑面板变成状态看板。
  - **验收：** 文案说明结果、原因和可行动作；中英文、深浅色、DPI 下可读，且提示关闭、重复操作和应用退出不会残留或抢焦点。

### 中期候选：有明确使用场景再做

- [ ] **日出日落偏移量配置**
  - **用户价值：** 支持比实际日出更早切浅色、比日落更晚切深色等个人习惯，减少用户为适应固定规则而手动切换。
  - **前置：** 先确认现有日出日落模式确有此类反馈；保持默认行为不变。

- [ ] **Day / Night “Scheduled / 已按计划”状态**
  - **用户价值：** 在用户确实困惑自动主题是否生效时，补充简短状态或 tooltip，降低猜测成本。
  - **前置：** 不在主面板堆叠常驻信息；优先用轻量提示验证。

### 保持现状或仅在真实反馈后调整

| 项目 | 推荐结论 | 用户价值与实施成本 | 风险、前置与触发条件 |
| --- | --- | --- | --- |
| quick / language hover 展开与 220ms 宽限 | **保持现状，不建议近期调整** | 当前只展开菜单、不直接执行命令，open 箭头和键盘/Escape 已清晰；改为点击优先的收益未被证实。 | 误触反馈或可用性测试明确显示问题时，再单独比较点击优先方案；不能破坏现有定位和键盘规则。 |
| panel padding 18→20、section gap 14→16、统一 4/8 spacing、section title 700→600 | **暂缓；仅独立视觉试验后决定** | 数字整齐本身不是价值；当前中英文截图密度正常，机械调整会让常驻工具更高、更松。 | 若出现阅读拥挤或本地化截断证据，再做一轮固定尺寸 A/B 截图和人工验收；不与功能改动混做。 |
| Tooltip 视觉统一/自绘 | **不建议实施** | 当前原生 `tooltips_class32` 已随主题、DPI 和窗口生命周期工作；圆角一致性不足以抵消收益。 | 自绘会增加 DPI、无障碍、定位和销毁风险；仅修复已复现的原生 tooltip 缺陷。 |
| hover 过渡、箭头翻转、choice layer 淡入、Toggle 动效 | **不建议实施** | 静态 open/pressed/focus 状态已足够清晰，默认无动画符合安静、低资源定位。 | 动画会引入 timer、CPU、减少动画偏好和关闭生命周期问题；仅在明确可用性收益下尝试 80–120ms、可关闭且非持续计时的局部效果。 |

### 低优先级：专项需求或质量问题触发

- [ ] **Windows High Contrast 支持**：服务特定辅助主题用户，但当前不作为主线体验投入；仅在明确用户需求或兼容问题出现时评估。
- [ ] **最小 UIA / MSAA 路径**：不为完整语义覆盖主动建设；只有收到读屏使用障碍反馈并能定位到具体控件时才补齐。
- [ ] **README 截图的 Windows CI artifact**：只增加内部验证证据，不能直接改善产品体验；有稳定发布流程需求时再试验，不做阻塞门禁。
- [x] **GDI+ / Direct2D 绘制后端研究**：当前 GDI + 局部 GDI+ 方案已满足需求；仅在出现可复现质量缺陷时继续评估，不主动迁移更多绘制内容。

### 明确过滤的通用设计建议

- 不采用 Web/移动端的 44–48pt 触控目标、响应式断点、React/Tailwind/WebView、网络字体或网页式卡片布局：它们不适用于现有 Windows 鼠标/键盘原生工具。
- 不为“现代感”引入玻璃、模糊、强阴影、渐变、发光、复杂动效或独立插画；这些会削弱 Quiet Native Utility 的克制性并增加 GDI 成本。
- 不把通用“全面无障碍框架”、自绘 tooltip 或绘制后端迁移当作近期必做项；均应以实测缺口或可复现质量问题为前置。

## 后续计划

### 排序判断

- **前置条件：** 性能和架构事项先给出当前症状、影响范围和可复现或可测量证据；没有问题证据，不因替换了更新的 Windows API、减少了少量轮询或代码更“正统”而排在用户功能之前。
- **可前置的技术项：** 能明显减少常驻耗电/CPU、修复状态迟滞或错误恢复、消除常见崩溃/托盘失效，或显著缩短用户完成操作的等待时间。
- **保持低优先级的技术项：** 主要为了架构整洁、API 现代化、测试覆盖、自动化方便，或只为无障碍语义完整而没有当前用户问题的改动。

### 高优先级：普通用户立刻受益

- [ ] **自动升级**：检查 GitHub Releases 最新稳定版本、下载对应架构 EXE、校验 SHA256，并在下次启动原子替换；把手动下载、覆盖 EXE 的负担从用户手中移走。
  - 需展示版本、大小、校验和与发布说明；失败可回滚，不自动跟随预发布版本，也不在未经用户选择时替换正在使用的 EXE。
- [ ] **空闲时长支持自定义值**：保留常用预设，同时允许输入任意分钟数；避免用户为了一个不在预设内的时长修改配置文件。
  - 需在界面中完成输入、范围校验和错误提示，并保留现有 `0 = 禁用` 语义；不能让无效输入静默写入配置。

### 中优先级：有反馈或使用场景再推进

- [ ] **自动升级的进度、结果与控制方式**：在浮层展示“检查更新 / 下载进度 / 下次启动完成升级”等明确状态，并提供简单的“仅检查、下载后询问、自动下载”选择；让更新可见、可控。先完成基础自动升级和安全回滚，再决定默认策略与细节。
- [ ] **托盘图标稳定 GUID 与 `NIM_SETVERSION`**：若 Explorer 重启、图标丢失或鼠标事件不一致有真实反馈，再提升通知区域恢复体验。
- [ ] **更多语言**：有明确用户群体和维护能力后再扩展，避免提前增加翻译负担。

### 低优先级：技术债、性能或内部验证

- [ ] **Windows 实机自动化冒烟测试**：仅提升内部回归效率，不直接增加用户功能；版本发布频率或回归成本明显上升时再投入。
- [ ] 用 `PowerCreateRequest/PowerSetRequest` 替换旧的线程执行状态 API；当前行为稳定时不为“正统化”而重构。
- [ ] 用 `RegisterPowerSettingNotification` 替换部分 3 秒电源轮询；若已观测到耗电、延迟或状态不同步，可按“排序判断”前置，否则不作为性能优化项目推进。
- [ ] 用 `ReadDirectoryChangesW` 替换配置自动重载的 `os.Stat` 轮询；只有实际轮询成本或外部编辑生效延迟成为问题时再做。
- [ ] CLI `status --json`：面向脚本和监控，不是普通用户主路径；有明确自动化消费者后再实现。
- [ ] 统一 quick/language/choice 浮层 metrics、清理 choice 临时像素补偿，或评估预创建 option HWND：仅在可见 UI 缺陷或可测量性能问题出现时处理。
- [ ] 空闲预警倒计时主动取消：当前序号校验已保证隐藏后的旧倒计时不会更新错误窗口；仅在长预警时间、频繁取消场景出现可测量的 goroutine 或 UI 消息积压时补充取消生命周期。
- [ ] Process Watch 回调 generation 防护：当前 watcher 停止会等待后台轮询退出；仅在复现旧回调晚到并覆盖新状态后增加代际标识，不为理论竞态增加同步复杂度。
- [ ] 继续拆分 `popup.build`、`popup.handleCommand`、`tray.handlePopupAction` 等顺序型大函数：只在相关功能再次修改、审查成本明显上升时按职责小步拆分，不为复杂度指标制造跨层抽象。
- [ ] 针对性启用 `errcheck`：先为 Win32 `Proc.Call`、幂等清理和刻意忽略返回值建立明确排除，再逐步覆盖普通错误路径；不直接全仓开启产生大量无效告警。
- [ ] 独立 Windows 7 legacy 构建：仅在真实用户需求出现时评估。
- [ ] CI race detector：仅测试增强，短期不投入。
