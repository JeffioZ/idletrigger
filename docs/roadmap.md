## 🎨 UI/UX 后续计划：Quiet Native Utility

本节只记录仍有价值、尚未完成的 UI/UX 工作。优先级以用户可感知的收益为准，例如减少操作、避免误触和解释状态。性能或架构工作只有在能解决可复现问题时才前置。保持现有信息架构，不引入 WebView、网页技术或大型 UI 框架。

### 已完成（不再列为待办）

- [x] controlpanel 资源生命周期、style / metrics / layout 集中，以及真实深浅色、中英文 UI 截图基线。
- [x] choice button + choice layer、触发器上下箭头和真实 open 状态、selected marker 与稳定字重。
- [x] 键盘 focus-visible 语义与圆角 focus ring、Toggle 的局部 hover / pressed 反馈、Shutdown / Restart 的危险分组与分隔线。
- [x] 主图标、深浅 tray/title 图标和 GitHub Social Preview 的统一品牌资源更新。

### 近期推荐：直接改善日常体验

- [ ] **Shutdown / Restart 的原生确认（需要产品确认）**
  - **价值：** 关机和重启前默认取消，降低误触损失；睡眠和休眠仍立即执行。
  - **方案：** 仅覆盖 Shutdown、Restart，使用轻量原生确认框。
  - **风险：** 会多一步操作。系统电源动作不可可靠撤销，因此不使用短暂撤销或二次点击。
  - **验收：** 鼠标和键盘均能安全取消/确认；默认焦点为取消；现有命令、语言、DPI、主题和菜单关闭生命周期不回归。

- [ ] **关键状态与失败原因的轻量反馈**
  - **价值：** 说明设置是否生效、为何暂停，以及下一步操作。
  - **范围：** 优先覆盖更新失败、全屏暂停主题切换，以及保持唤醒与空闲监测互斥。
  - **边界：** 只在用户操作、失败或需要处理时显示短提示；成功时不堆积通知。
  - **验收：** 文案包含结果、原因和可执行操作；中英文、主题和 DPI 下均清晰，且不残留或抢焦点。

### 中期候选：有明确使用场景再做

- [ ] **配置能力的渐进式界面补齐**
  - **定位：** 现有控制面板与 TOML 已覆盖日常使用；以下项目不阻塞当前使用，仅在有空或收到明确反馈时逐项完善，不要求一次完成。
  - **保持唤醒：** 为手动设置补充“屏幕常亮”“电池供电时允许”和低电量停用阈值；自动任务已有的屏幕常亮选项保持独立。
  - **昼夜模式：** 按需补充 fixed / sunrise 模式、浅色/深色时间和手动经纬度编辑；保留现有 IP 定位、电池深色、全屏暂停与计划摘要。
  - **空闲监测：** 在常用时长预设之外允许输入自定义分钟数，并允许调整执行前提醒秒数；继续保留 `0 = 禁用` 及现有范围校验。
  - **语言：** 在 English / 简体中文之外补回“跟随系统（auto）”入口。
  - **边界：** 优先采用渐进展示或独立设置面板，避免挤满主控制浮层；每项需同时覆盖中英文、深浅色、DPI、配置保存与重载，不暴露自动任务内部 ID 等实现字段。

- [ ] **日出日落偏移量配置**
  - 支持在日出前或日落后切换，减少手动调整。
  - 先确认真实需求，并保持默认行为不变。

- [ ] **Day / Night “Scheduled / 已按计划”状态**
  - 用户不确定自动主题是否生效时，显示简短状态或 tooltip。
  - 不增加常驻信息，先用轻量提示验证。

### 保持现状或仅在真实反馈后调整

- **quick / language 悬停与 220ms 宽限：保持现状。** 当前只展开菜单，不执行命令；箭头、键盘和 Escape 已清晰。收到明确误触反馈后再比较点击方案。
- **间距和字重微调：暂缓。** 当前中英文密度正常。只有出现拥挤或截断时，才单独进行固定尺寸 A/B 截图测试。
- **Tooltip 自绘：不实施。** 原生 tooltip 已支持主题和 DPI。仅修复可复现的原生缺陷。
- **悬停、箭头、浮层和 Toggle 动画：不实施。** 静态状态已经清晰。只有明确改善可用性时，才测试可关闭的短动画。

### 低优先级：专项需求或质量问题触发

- [ ] **Windows High Contrast 支持**：有明确需求或兼容问题时再评估。
- [ ] **最小 UIA / MSAA 路径**：收到具体读屏问题后再补齐对应控件。
- [ ] **README 截图的 Windows CI artifact**：发布验证确有需要时再试验，不作为阻塞门禁。
- [x] **GDI+ / Direct2D 绘制后端研究**：当前方案已满足需求；只在出现可复现缺陷时继续评估。

### 明确过滤的通用设计建议

- 不照搬 Web/移动端规范，不引入 React、Tailwind、WebView、网络字体或网页式布局。
- 不为“现代感”加入玻璃、模糊、强阴影、发光、复杂动画或独立插画。
- 不把完整无障碍框架、自绘 tooltip 或绘制后端迁移列为近期任务；先确认具体问题。

## 🧭 后续计划

### 排序判断

- **先给证据：** 性能和架构事项需说明症状、影响范围和复现或测量结果。
- **可以前置：** 明显降低耗电或 CPU，修复状态延迟、恢复失败、常见崩溃或托盘失效。
- **保持低优先级：** 仅为了架构整洁、API 更新、测试覆盖或自动化方便的改动。

### 高优先级：普通用户立刻受益

- [ ] **自动升级**：检查最新稳定版，下载对应架构，校验 SHA-256，并在下次启动时原子替换。
  - 展示版本、大小、校验和与发布说明。失败可回滚，不跟随预发布版，也不擅自替换正在运行的 EXE。

### 中优先级：有反馈或使用场景再推进

- [ ] **自动升级的进度与控制**：显示检查、下载和待安装状态，并提供“仅检查、下载后询问、自动下载”。基础升级和回滚完成后再决定默认策略。
- [ ] **托盘图标稳定 GUID 与 `NIM_SETVERSION`**：出现 Explorer 重启、图标丢失或鼠标事件问题后再实施。
- [ ] **更多语言**：有明确用户群体和维护能力后再扩展，避免提前增加翻译负担。

### 低优先级：技术债、性能或内部验证

- [ ] **Windows 实机自动化冒烟测试**：发布频率或回归成本明显上升时再投入。
- [ ] 用 `PowerCreateRequest/PowerSetRequest` 替换线程执行状态 API：现有行为稳定时不重构。
- [ ] 用 `RegisterPowerSettingNotification` 替换部分电源轮询：仅在出现耗电、延迟或状态不同步后实施。
- [ ] 用 `ReadDirectoryChangesW` 替换配置轮询：仅在轮询成本或生效延迟成为问题后实施。
- [ ] CLI `status --json`：面向脚本和监控，不是普通用户主路径；有明确自动化消费者后再实现。
- [ ] 统一浮层 metrics、清理像素补偿或预创建 HWND：仅在出现可见缺陷或可测性能问题后处理。
- [ ] 主动取消空闲预警倒计时：仅在长倒计时或频繁取消造成可测积压后实施。
- [ ] 拆分控制面板大函数：只在相关功能再次修改且审查成本上升时小步进行。
- [ ] 针对性启用 `errcheck`：先定义 Win32 调用和刻意忽略返回值的排除规则，再逐步覆盖。
- [ ] 独立 Windows 7 legacy 构建：仅在真实用户需求出现时评估。
- [ ] CI race detector：仅测试增强，短期不投入。
